A given program has $25 \%$ load/store instructions. Suppose the ideal $\text{CPI}$ (cycles per instruction) without any memory stalls is $2$. The program exhibits $2 \%$ miss rate on instruction cache and $8 \%$ miss rate on data cache. The miss penalty is $100$ cycles. The speedup (rounded off to two decimal places) achieved with a perfect cache (i.e., with NO data or instruction cache misses) is __________.

CPI with a Perfect Cache: $2$

CPI with the Actual Cache: 2 (ideally) + $0.02 \times 100$ (stall cycles for instruction cache miss) + $0.25 \times 0.08 \times 100$ (stall cycles for data cache miss) = $6$

Speedup with a perfect cache = $6/2 = 3.$

Answer : 3

Let's Assume there are 100 instructions,

so we have 100 fetches for 100 instructions, this will come from instruction cache.

25% that is 25 instructions are Data fetch / store instructions - come from data cache

so in total 100 fetches we will have 2% ie, 2 misses - 200 cycles penality.

from the data stream, 8% ie, 2 misses - 200  cycles penality.

Total cycles taken to execute 100 instructions is 100*2 + 200 + 200 = 600 cycles

CPI = 600/100 = 6.

In a perfect caching solution we have 0 misses and hence the CPI will be default ie, 2 CPI.

Speedup achieved = 6/2 = 3

BASIC APPROACH (For Understanding )

Let Instruction Cache access time be =x1 cycle,
&    Data Cache acess time be =x2 cycle,
& assume total instructions in program =100.
So,

25 inst are load/store i.e. they access (data cache + instr cache) for each 25 instructions, &
Remaining 75 instructions will access only inst cache.
so overall you can say you are going to access Instr cache for 100times(25+75) AND Data cache for 25times

Ideally (with no misses neither in Inst cache nor in Data cache) :

Total time in accessing Inst Cache=100 * x1 cycle ,
Total time in accessing Data Cache=25 * x2 cycle
Total Time in accesing 100 instructions= (100 * x1) +(25 * x2) cycles ......................(i)

Also given that ideally for each inst it takes 2 cycles i.e CPI=2
so, For 100 inst , total accesing time ideally= 100*2 =200cycles.............................................................(ii)

equating eqn1 and 2
100*x1 +25*x2 = 200...................(iii)

Non-Ideal:
Actually lets see how much it takes  non ideallly :
Remember we r accesing ins cache for 100 times and data cache for 25 times,

on an avg 1 inst in Instr cache takes=cache access time + miss rate * miss penalty
=x1 + 0.02 * 100 cycles

For 100 times , Toatal Inst cache access time =100 *(x1 + 0.02*100)= 100*x1 +200 cycles

sismilarly , on an avg 1 inst in Data cache takes= cache access time + miss rate *miss penalty
= x2 + 0.08 *100 cycles

For 25 times ,Total Data cache access times=25*(x2+0.08*100)=25*x2 +200

Total time taken for 100 inst= total inst cache time + total data cache time
=(100*x1 + 200) + (25*x2 +200) =100*x1 +25*x2 +400 cycles

using equation (iii) we get
Toatal accessing time for 100 inst=200+400=600 cycles

Speedup (or Performance gain) = (total accessing time non-ideal) / (total accesssing time ideally)

Speedup=600/200=3
