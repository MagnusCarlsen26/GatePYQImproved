Arjun asked Sep 23, 2014 • retagged Nov 13, 2017 by Arjun
20,955 views

In a $k$-way set associative cache, the cache is divided into $v$ sets, each of which consists of $k$ lines. The lines of a set are placed in sequence one after another. The lines in set $s$ are sequenced before the lines in set $(s+1)$. The main memory blocks are numbered 0 onwards. The main memory block numbered $j$ must be mapped to any one of the cache lines from

  1. $(j\text{ mod }v) * k \text{ to } (j \text{ mod } v) * k + (k-1) $
  2. $(j \text{ mod } v) \text{ to } (j \text{ mod } v) + (k-1) $
  3. $(j \text{ mod } k) \text{ to } (j \text{ mod } k) + (v-1) $
  4. $(j \text{ mod } k) * v \text{ to } (j \text{ mod } k) * v + (v-1) $

Number of sets in cache $= v$.

The question gives a sequencing for the cache lines. For set $0$, the cache lines are numbered $0, 1, .., k-1$. Now for set $1$, the cache lines are numbered $k, k+1,... k+k-1$ and so on.

So, main memory block $j$ will be mapped to set $(j \ \text{mod} \ v)$, which will be any one of the cache lines from $(j \ \text{mod  } v) * k \ \text{ to } (j \ \text{mod  } v) * k + (k-1)$.
(Associativity plays no role in mapping- $k$-way associativity means there are $k$ spaces for a block and hence reduces the chances of replacement.)

Correct Answer: $A$

Think of it as two-dimensional array, $A[v][k]$ with $v$ rows and $k$ columns since each set $i$ has exactly $k$ blocks in it.

A block $j$ will map to set $j\%v$ and any of the blocks $0$ through $k-1$ of that set.

Thus, you need the position o f $A[j\%v][0]$ and $A[j\%v][k-1]$ in memory. Using simple 2D array address translation, these come out to be $(j\%v)*k$ and $(j\%v)*k+(k-1)$.

Answer: Option (A) .

Set Mapping:

In a k-way set associative cache, main memory blocks are mapped to specific sets within the cache.
The set number is determined by taking the modulo of the block number with the number of sets (v).
So, block number j maps to set number (j mod v).

Line Placement Within a Set:

Each set contains k lines, and they are placed sequentially within the set.
The first line of set s begins at index (s * k), and the last line of set s is at index (s * k + k - 1).

Mapping Block j:

Block j maps to set (j mod v), so it can be placed in any line within that set.
The range of lines for set (j mod v) is from ((j mod v) * k) to ((j mod v) * k + k - 1).
Therefore, the main memory block numbered j must be mapped to any one of the cache lines from (j mod v) * k to (j mod v) * k + (k-1).

Analysis

1. Find the Target Set Number

In a set-associative cache with v sets, a main memory block numbered j is mapped to a cache set using the modulo operator.

  • Set Number = j mod v

2. Find the Line Numbers for that Set

The problem states that the cache lines are numbered sequentially. Each of the v sets contains k lines.

  • Set 0 contains the first k lines, which are numbered: 0, 1, ..., k-1 .

  • Set 1 contains the next k lines, which are numbered: k, k+1, ..., 2k-1 .

  • Set 2 contains the next k lines, numbered: 2k, 2k+1, ..., 3k-1 .

Following this pattern, for any given set s , the first line number is s * k . Since there are k lines in the set, the last line number is s * k + (k - 1) .

3. Combine the Results

  • Let s = j mod v .

  • Starting Line: s * k becomes (j mod v) * k .

  • Ending Line: s * k + (k - 1) becomes (j mod v) * k + (k - 1) .

This range of cache lines matches option A perfectly.
