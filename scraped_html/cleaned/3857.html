GATE IT 2005 | Question: 84b

Ishrat Jahan asked Nov 3, 2014 • edited Oct 25, 2017 by kenzou
10,916 views
48 48 votes

A sink in a directed graph is a vertex i such that there is an edge from every vertex $j \neq i$ to $i$ and there is no edge from $i$ to any other vertex. A directed graph $G$ with $n$ vertices is represented by its adjacency matrix $A$, where $A[i] [j] = 1$ if there is an edge directed from vertex $i$ to $j$ and $0$ otherwise. The following algorithm determines whether there is a sink in the graph $G$. 

i = 0;
do {
    j = i + 1;
    while ((j < n) && E1) j++;
    if (j < n) E2;
} while (j < n);
flag = 1;
for (j = 0; j < n; j++)
    if ((j! = i) && E3) flag = 0;
if (flag) printf("Sink exists") ;
else printf ("Sink does not exist");

Choose the correct expression for $E_3$

  1. $(A[i][j] \ \&\& \ !A[j][i])$
  2. $(!A[i][j] \ \&\& \ A[j][i])$
  3. $(!A[i][j] \ \left | \right | A[j][i])$
  4. $(A[i][j] \ \left | \right | \ !A[j][i])$

If there is a sink in the graph, the adjacency matrix will contain all 1s (except diagonal) in one column and all 0s (except diagonal) in the corresponding row of that vertex. The given algorithm is a smart way of doing this as it finds the sink in $O(n)$ time complexity.

The first part of the code, is finding if there is any vertex which does not have any outgoing edge to any vertex coming after it in adjacency matrix. The smart part of the code is $E_2$, which makes rows skip when there is no edge from i to it, making it impossible them to form a sink. This is done through

$E_1:  \ !A[i][j]$
and
$E_2: i = j$;

$E_1$ makes sure that there is no edge from $i$ to $j$ and $i$ is a potential sink till $A[i][j]$ becomes $1$. If $A[i][j]$ becomes $1$, $i$ can no longer be a sink, similarly all previous $j$ can also not be a sink (as there was no edge from $i$ to them and a sink requires an edge from all other vertices). Now, the next potential candidate for sink is $j$. So, in $E_2$, we must make $i = j$.

Now, the loop breaks when we found a potential sink- that is a vertex which does not have any outgoing edge to any coming after it in adjacency matrix. So, if the column in which this vertex comes is all 1s and the row is all 0s (except diagonal), this is the sink. Otherwise there is no sink in the graph. So, $E_3$ is checking this condition.

But in the code $\text{flag}$ is used for storing the state that sink is present or not. And as per the usage of flag in code, by default sink is considered present.  So, the condition in $E_3$  must make flag = 0, if the found $i$ is not a sink. So, the condition should be:

$A[i][j] \  \left | \right |\ !A[j][i]$

So, (D) is the answer .

Important point where we can make mistake.


we know given Adjacency Matrix to check if any vertex is a sink or not :

1. Its row must be all zeros (no outgoing edges).

2.Its column must be all ones except the diagonal (incoming from all others).


A B C D A 0110 B 1110 C 0000 D 1111    Like here C is a sink.

After while loop execution, i=2 (acc. to above example) since all row entries of vertex C are 0, so its our potential candidate for sink as of now.

Now, Interesting point, since this probelm is of Identifying the sink vertex so our mind will think that E3 condition will be there to verify whether C is really a sink vertex or not and which is a trap.


Because if you see flag is alredy set to 1 even before for loop execution which tells question already assumed that pontential candidate as sink vertex , while inside for loop that E3 condition is to set flag=0 i.e E3 is actually the failure condition i.e it checks if sink does not exist  (A [ i ] [ j ]   | |   ! A [ j ] [ i ] )

and clearly option (D) is that condition.
If we flip the flag values then answer will become option (B) because then E3 will check if sink exist.

We are given a directed graph with $ n $ vertices represented by an adjacency matrix $ A $, where $ A[i][j] = 1 $ if there is an edge from vertex $ i $ to vertex $ j $, and $ 0 $ otherwise. A sink is a vertex $ i $ such that:

There is an edge from every vertex $ j \ne i $ to $ i $: $ A[j][i] = 1 $ for all $ j \ne i $,
There is no edge from $ i $ to any other vertex: $ A[i][j] = 0 $ for all $ j \ne i $.
The algorithm first identifies a candidate sink vertex $ i $ using the loop with expressions $ E_1 $ and $ E_2 $ (as previously determined: $ E_1 = \texttt{!A[i][j]} $, $ E_2 = \texttt{i = j} $). After identifying the candidate $ i $, it verifies whether $ i $ is indeed a sink using a second loop:

flag = 1;
for (j = 0; j < n; j++)
if ((j != i) && E3) flag = 0;
if (flag) printf("Sink exists");
else printf("Sink does not exist");
Here, $ E_3 $ must be the condition that, if true, indicates that the candidate $ i $ fails to be a sink.

What must $ E_3 $ check?

For $ i $ to be a sink, two conditions must hold for every $ j \ne i $:

No outgoing edge from $ i $: $ A[i][j] = 0 $
Incoming edge to $ i $: $ A[j][i] = 1 $
If either condition fails for any $ j \ne i $, then $ i $ is not a sink.

Therefore, $ E_3 $ should be true if either:

$ A[i][j] = 1 $ (violation of no outgoing edges), or
$ A[j][i] = 0 $ (violation of incoming edges from all others).
In logical terms:

$ E_3 : A[i][j] = 1 \text{ or } A[j][i] = 0 $

In C syntax, this is:

$ E_3 : (A[i][j] \ || \ !A[j][i]) $

This matches option D.

$$
\boxed{\text{D. } (A[i][j] \ || \ !A[j][i])}
$$
