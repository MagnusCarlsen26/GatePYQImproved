A non-pipelined instruction execution unit operating at $2 \mathrm{GHz}$ takes an average of $6$ cycles to execute an instruction of a program $\text{P}$. The unit is then redesigned to operate on a $5$ -stage pipeline at $2 \mathrm{GHz}$. Assume that the ideal throughput of the pipelined unit is $1$ instruction per cycle. In the execution of program $\text{P}$, $20 \%$ instructions incur an average of $2$ cycles stall due to data hazards and $20 \%$ instructions incur an average of $3$ cycles stall due to control hazards. The speedup (rounded off to one decimal place) obtained by the pipelined design over the non-pipelined design is ____________.

Tn = 1/2 GHz  =  0.5 ns
Execution Time non- pipe = Tn * CPI  =  0.5 * 6  =  3 ns

Tp = 1/2 GHz  =  0.5 ns
20%  causes 2 stalls  and  20% causes 3 stalls
stalls per instr.  =  0.2*2 + 0.2*3 = 1
Execution Time pipe = (1 + stalls per instr) * Tp = (1 + 1) * 0.5 = 1 ns

Speed-up = Exec non-pipe / Exec pipe  =  3 / 1  = 3

Non-pipelined Design:
A non-pipelined instruction takes $6$ cycles to execute an instruction.
Average CPI = $6$

Pipelined Design:
It is given that, $20\%$ of instructions incur $2$ cycles stall and $20\%$ of instructions incur $3$ cycles stall.
Average CPI $= 1 + (0.2*2) + (0.2*3) = 1+0.4+0.6 = 2$

Speedup = $\frac{\text{Avg CPI in NPL } * \cancel{CycleTime_{NPL}}}{\text{Avg CPI in PL } * \cancel{CycleTime_{PL}}} = \frac{\text{6}}{\text{2}} = 3$

Since, frequency of both design is same, cycle time will also be same. So, I cut off cycle time from numerator and denominator.

Note:
In pipelined design, 1 cycle is taken by ALL instructions but 20% of them are taking 2 Stall cycles, which means they are taking "extra" 2 cycles & similarly other 20% of them are taking Extra 3 cycles.
2.9 to 3.1 is a correct answer. (Acc to Official Answer key)

As per the question given

Non-Pipeline:
Tnp = (1/2Gz) = 0.5 nsec
CPI(Non-pipeline) = 6 cycles

Same goes with Pipeline:
Tp = (1/2Gz) = 0.5 nsec
CPI(Pipeline) = 1(Ideal Condition) + No.of Stalls per instruction => 1 + [0.2*2 + 0.2*3 + 0.6*0] => 1 + [0.4+0.6] => 1+1 = 2 cycles

The Speedup Factor with Stalls is :

S(Speedup) = ETnp / ETp = [CPI(non-pipe) * Tnp] / [CPI(pipe) * Tp] = [6*0.5] / [2*0.5] = 3.0

Assume that the ideal throughput of the pipelined unit is 1 instruction per cycle

The throughput of a pipelined processor is simply the number of instructions coming out from the last stage of those 5 stages per unit time. In our case the time unit is 1 clock cycle which is 0.5 nsec,

which means each stage in the pipeline processor takes only 1 clock cycle to operate on. ( ignoring the register delay and clock skew)

and the CPI is defined as average clock cycles per instruction , meaning , it takes 6 clock cycles to complete an instruction under a non pipelined processor but takes only 1 clock cycle on an average to complete an instruction under a piplined processor.

CPI(non-pipelined)  = 6

CPI(pipelined)         = 1

Speed up = CPI(non-pipelined) / ideal CPI(pipelined) + pipeline stall clock cycles

we are adding the ideal CPI with pipeline stall cycle becoz clock cycles get wasted during stall of some instrcuctions

thus Speedup = IC x 6 x 1 clock cycle /IC x(1+ 20% x 2 + 20% x 3) x 1 clock cycle time

( IC =  total instruction count of program P )

Thus Pipelined processor is 3.0 times faster than the non pipelined processor.
