In a $\mathrm{B}^{+}$- tree where each node can hold at most four key values, a root to leaf path consists of the following nodes:
\[
\mathrm{A}=(49,77,83,-), \mathrm{B}=(7,19,33,44), \mathrm{C}=\left(20^{*}, 22^{*}, 25^{*}, 26^{*}\right)
\]

The $*$-marked keys signify that these are data entries in a leaf.

Assume that a pointer between keys $k_{1}$ and $k_{2}$ points to a subtree containing keys in $\left[k_{1}, k_{2}\right)$, and that when a leaf is created, the smallest key in it is copied up into its parent.

A record with key value $23$ is inserted into the $\mathrm{B}^{+}$- tree.

The smallest key value in the parent of the leaf that contains $25^{*}$ is __________. (Answer in integer)

We are given a B⁺ tree where each node can hold at most four key values. The initial root-to-leaf path consists of:

  • $A = (49, 77, 83, -)$  internal node
  • $B = (7, 19, 33, 44)$ internal node
  • $C = (20^*, 22^*, 25^*, 26^*)$ leaf node (keys marked with * are data entries)

Assumptions:

  • A pointer between keys $k_1$ and $k_2$ points to a subtree containing keys in the interval $[k_1, k_2)$.
  • When a leaf is created due to a split, the **smallest key** in that new leaf is copied up into its parent.

We insert key value $23$, and are to find:The smallest key value in the parent of the leaf that contains $25^*$ after insertion.

Step 1: Insert key 23 into leaf C

Leaf $C = (20^*, 22^*, 25^*, 26^*)$ → inserting $23$ yields:

$$
(20^*, 22^*, 23^*, 25^*, 26^*)
$$

This exceeds capacity (max 4 keys) → leaf must split.

Step 2: Split leaf C (using problem’s rule: promote smallest key of new right leaf)

Split into:

  • Left leaf: $(20^*, 22^*)$
  • Right leaf: $(23^*, 25^*, 26^*)$

Smallest key in the right leaf is $23^*$ → copy $23$ up to parent node B.

Step 3: Insert 23 into parent B

Node B was: $(7, 19, 33, 44)$

After inserting 23: $(7, 19, 23, 33, 44)$ → now has 5 keys → exceeds capacity → must split.

Step 4: Split node B (internal node)

With 5 keys: $(7, 19, 23, 33, 44)$

Promote median key $23$ to parent A.

Split into:

  • Left child: $(7, 19)$
  • Right child: $(33, 44)$

Step 5: Update node A

Node A was: $(49, 77, 83, -)$

Insert $23$: becomes $(23, 49, 77, 83)$ — still within capacity (≤4 keys) → no further split.

Step 6: Determine parent of leaf containing $25^*$

The leaf $(23^*, 25^*, 26^*)$ is attached to the **right child of B**, which is now $(33, 44)$.

Thus, the parent of the leaf containing $25^*$ is the node $(33, 44)$.

The parent node is $(33, 44)$, so the smallest key in it is:

$$
33
$$

Few things to note:

  1. When a leaf is (splitted from original leaf) created, the smallest key in it is copied up into its immediate parent.(i.e its present as smallest key in leaf AND in sorted position in parent of that leaf) (splitting in B+ tree)
  2. When an internal node is splitted then no need of keeping the middle element in both new splitted node and in parent of the node, we keep middle element just in parent (splitting in B tree)
