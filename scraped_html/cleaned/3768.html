For those who are finding it difficult to understand for the 1st time:

All levels and all nodes are completely filled with keys.
Another key came.
It went down to leaf.
Overflow happened at the leaf.
The leaf is broken into two parts and one key moved to the upper level.
Again overflow. That node is broken into two parts and one key moved to the upper level.
Same thing will happen till we reach the root.
The root itself is broken into two parts + we have a new root.

So in all 'n' levels, we got one extra node and we got a new root too.
So total n+1 extra nodes.  Hence answer is 4+1 = 5.

A

5 is the correct answer. To find maximum no of nodes newly created, assume that all nodes are full. Adding one more to leaf will split the leaf at level 4 (+1) , the middle element moves up and splits the parent node at level 3 (+2), similiarly a split at level 2(+3) , and now at root node a split (+4) and its middle element forms a new root node(+5). Now the B tree has 5 Levels with 5 new nodes created

Assume all the levels are completely filled.
so at level 1 (root) we have assume n children nodes.
At level 2 we have n children nodes each containing n nodes.
similarly for level 3 and level 4.
Now assume a new node is to be inserted and it is inserted in the last node of the last level.
Since last level is already full and when we insert the element it will cause overflow, so to avoid avoid overflow we cause splitting at the last level , at the last node, thus creating a new node for accommodating the new node we are inserting.
Now when we split the last node , it's mid element will be pushed on to the parent (level 3 last node). Now to accommodate this node we again have to split the node thus creating 1 more node, similarly its mid node will be pushed on to its parent, thus creating one more split.
This will happen till we reach the level 1, now level 1 node will again be splitted to accommodate this newly created node.
Thus total new nodes required are n levels +1 that is 4+1.
