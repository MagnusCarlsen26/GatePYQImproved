GATE CSE 2025 | Set 2 | Question: 28

Arjun asked Feb 27, 2025 • edited Mar 28, 2025 by Misbah Ghaya
6,345 views
19 19 votes

​A meld operation on two instances of a data structure combines them into one single instance of the same data structure. Consider the following data structures:

P. Unsorted doubly linked list with pointers to the head node and tail node of the list.

Q. Min-heap implemented using an array.

R. Binary Search Tree.

Which ONE of the following options gives the worst-case time complexities for meld operation on instances of size $n$ of these data structures?

  1. $\mathrm{P}: \Theta(1), \mathrm{Q}: \Theta(n), \mathrm{R}: \Theta(n)$
  2. $\text{P}$: $\Theta(1), \mathrm{Q}: \Theta(n \log n), \mathrm{R}: \Theta(n)$
  3. $\text{P}$: $\Theta(n), \mathrm{Q}: \Theta(n \log n), \mathrm{R}: \Theta\left(n^{2}\right)$
  4. $\text{P}$: $\Theta(1), \mathrm{Q}: \Theta(n), \mathrm{R}: \Theta(n \log n)$

The correct answer is A because melding two unsorted doubly linked lists takes O(1) time by simply adjusting a few pointers. For min-heaps implemented as arrays, we can concatenate the two heaps and run a single build-heap , which takes O(n) time. For binary search trees, the optimal meld method is to perform in-order traversal on both trees, merge the resulting sorted lists, and build a balanced BST from the merged list, all of which together also take O(n) time.

When we state the worst-case time complexity of an operation like meld , we refer to the worst-case running time of the best possible correct algorithm for that operation not a naive or inefficient implementation. This is the standard convention in algorithm analysis

1. Unsorted Doubly Linked List (with head and tail pointers)

Melding is done by linking the tail of the first list to the head of the second. With direct access to both ends, this requires only a constant number of pointer updates.
Time complexity: $ \Theta(1) $.

2. Min-Heap (array-based)

Although binary heaps do not natively support efficient melding, we can:

  • Concatenate the two underlying arrays (size $ n + n = 2n $),

  • Run the linear-time build-heap procedure on the result.

Since build-heap runs in $ \Theta(n) $ time for $ n $ elements, the total cost is $ \Theta(n) $.
Time complexity: $ \Theta(n) $.

3. Binary Search Tree (BST)

Meld does not require preserving the original tree structures only that the result is a valid BST containing all elements.

An optimal $ \Theta(n) $ algorithm exists:

  1. Perform in-order traversals of both trees to obtain two sorted arrays (each in $ \Theta(n) $ time).

  2. Merge the two sorted arrays into one (standard merge, $ \Theta(n) $).

  3. Construct a balanced BST from the merged sorted array by recursively picking the middle element as the root. This step also runs in $ \Theta(n) $ time and avoids degenerate (skewed) trees.

Optional Proof of $\theta (n)$:
The algorithm:

  • Picks the middle element → $ O(1) $ work,

  • Recursively builds left subtree from the left half (size $ \lfloor (n-1)/2 \rfloor $),

  • Recursively builds right subtree from the right half (size $ \lceil (n-1)/2 \rceil $).

So the recurrence is:

$$
T(n) = T\left(\left\lfloor \frac{n-1}{2} \right\rfloor\right) + T\left(\left\lceil \frac{n-1}{2} \right\rceil\right) + c
$$

for some constant $ c > 0 $, and base case:

$$
T(0) = d \quad \text{(constant time for empty subtree)}.
$$

For simplicity (and without loss of generality for asymptotics), assume $ n = 2^k - 1 $ (so the tree is perfectly balanced). Then:

$$
T(n) = 2T\left(\frac{n-1}{2}\right) + c.
$$

But even cleaner: since $ n-1 \approx n $ for large $ n $, we approximate:

$$
T(n) = 2T\left(\frac{n}{2}\right) + c. \tag{1}
$$

This is a standard recurrence.

As established, the time to build a BST from a sorted array of size $ n $ satisfies:

$$
T(n) = 2T\left(\frac{n}{2}\right) + c,
$$

where $ c > 0 $ is a constant (the work done at each recursive call: computing the middle index, allocating a node, etc.), and we assume $ n $ is a power of 2 for simplicity (standard in asymptotic analysis).

This is a standard divide-and-conquer recurrence of the form:

$$
T(n) = a,T\left(\frac{n}{b}\right) + f(n),
$$

with:

  • $ a = 2 $ (number of subproblems),

  • $ b = 2 $ (each subproblem is half the size),

  • $ f(n) = c = \Theta(1) $ (non-recursive work).

Master Theorem (Case Analysis)

First, compute the critical exponent:

$$
n^{\log_b a} = n^{\log_2 2} = n^1 = n.
$$

Now compare $ f(n) = \Theta(1) $ with $ n^{\log_b a} = n $.

We see that:

$$
f(n) = \Theta(1) = O\left(n^{\log_b a - \varepsilon}\right) \quad \text{for } \varepsilon = 1 > 0,
$$

since $ n^{1 - 1} = n^0 = 1 $, and $ \Theta(1) = O(1) $.

This matches Case 1 of the Master Theorem:

Case 1: If $ f(n) = O\left(n^{\log_b a - \varepsilon}\right) $ for some $ \varepsilon > 0 $, then
$$
T(n) = \Theta\left(n^{\log_b a}\right).
$$

Applying it:

$$
T(n) = \Theta\left(n^{\log_2 2}\right) = \Theta(n).
$$

By the Master Theorem, the recurrence

$$
T(n) = 2T\left(\frac{n}{2}\right) + \Theta(1)
$$

solves to:

$$
\boxed{T(n) = \Theta(n)}.
$$

Note: If one were to insert elements one-by-one into a BST in sorted order, the result would be a skewed tree and the cost would be $ \Theta(n^2) $. However, that is a suboptimal algorithm. Since a correct $ \Theta(n) $ algorithm exists, the worst-case time complexity of the meld operation itself is $ \Theta(n) $.

Time complexity: $ \Theta(n) $.

Final Matching

  • P (linked list): $ \Theta(1) $

  • Q (min-heap): $ \Theta(n) $

  • R (BST): $ \Theta(n) $

This corresponds to Option A.

Q. By converting two min heap instances into a single array and then apply BUILD-Min heap algorithm which internally uses heapify procedure. In general heapify procedure takes logn time and total time nlogn but by using sum of heights of all nodes approach it takes θ(n) time.

R. By applying in order traversal on two BST, we get elements in increasing order and then merge them into signle sorted risk takes θ(n) and construct BST with θ(n) time.

Because we are considering the worst case , the BST should be treated as highly skewed , and the insertion method should be assumed to be the naive BST insertion , not an optimized approach. In a highly skewed BST, the height is O(n) . Therefore:

  • Each insertion into the BST takes O(n) time.

  • If we perform nnn such insertions, the total time becomes O(n^2) .

So, in the worst case, the meld operation for BSTs takes O(n^2) time.

P. Unsorted doubly linked list with head and tail pointers

  • Since it’s unsorted , we don’t need to reorder anything.

  • Meld = just link the tail of the first list to the head of the second list and update tail pointer.

  • That’s Θ(1) .

P = Θ(1)

Q. Min-heap implemented using an array

  • If we have two heaps of size n , melding means combining them into one valid heap.

  • Best way: concatenate the arrays (size = 2n) and run BuildHeap .

  • BuildHeap runs in Θ(n) time (linear, not n log n).

Q = Θ(n)

R. Binary Search Tree (BST)

  • Two BSTs of size n each. To meld:

    1. Traverse both (in-order) ⇒ Θ(n) each.

    2. Merge two sorted arrays ⇒ Θ(n) .

    3. Build balanced BST from merged sorted array ⇒ Θ(n) .

So total = Θ(n) , not Θ(n²).

  • Θ(n²) would happen only if we insert nodes one by one without balancing, but the question asks for most time-efficient implementation , so we take Θ(n).

R = Θ(n)

Correct Option: A. P : Θ(1), Q : Θ(n), R : Θ(n)
