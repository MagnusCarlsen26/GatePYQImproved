The packet destination address is $200.150.68.118$. The router performs Longest Prefix Match (LPM) against its forwarding table entries to determine the correct interface.We compute the network ID for each entry by applying the bitwise AND operation between the destination IP and the subnet mask. Only entries whose computed network ID matches the given subnet number are considered valid.

$$
\begin{array}{|c|c|c|}
\hline
\text{Subnet Number} & \text{Subnet Mask} & \text{Interface ID} \\
\hline
200.150.0.0 & 255.255.0.0 & 1 \\
200.150.64.0 & 255.255.224.0 & 2 \\
200.150.68.0 & 255.255.255.0 & 3 \\
200.150.68.64 & 255.255.255.224 & 4 \\
\text{Default} & - & 0 \\
\hline
\end{array}
$$

Entry 1: $200.150.0.0/16$

  • Network ID = $200.150.68.118 \land 255.255.0.0 = 200.150.0.0$
  • Matches subnet number → Valid
  • Prefix length: 16 bits

Entry 2: $200.150.64.0/19$

  • Network ID = $200.150.68.118 \land 255.255.224.0 = 200.150.64.0$
  • Matches subnet number → Valid
  • Prefix length: 19 bits

Entry 3: $200.150.68.0/24$

  • Network ID = $200.150.68.118 \land 255.255.255.0 = 200.150.68.0$
  • Matches subnet number → Valid
  • Prefix length: 24 bits

Entry 4: $200.150.68.64/27$

  • Network ID = $200.150.68.118 \land 255.255.255.224 = 200.150.68.96$
  • Does not match $200.150.68.64$ → Invalid

Default: Always matches, but only if no longer prefix exists.

Longest Prefix Match Selection

Among valid entries, we select the one with the longest prefix:

/16 → Interface 1
/19 → Interface 2
/24 → Interface 3 ← Longest

Final Decision

The packet will be forwarded to the interface with ID

$$
\boxed{3}
$$

This result follows from the standard routing behavior of selecting the most specific matching route via longest prefix match.

To solve this question we have to use longest prefix match method
Given Destination IP address = 145.36.109.70

As we know that DIP AND Subnet mask = subnet Id (subnet address)

So we will simply perform bitwise AND on DIP and Subnet mask and match is the answer.

For 1st :

since number of 1's in subnet mask = 16 our subnet mask for this will be 255.255.0.0
So simply perform Bitwise AND on this 

145.36.109.70
AND 
255.255.0.0
=145.36.0.0

= 145.36.0.0..... It matches the subnet address 145.36.0.0 so is E1 the answer ?? No we have to check all the options as we need to find the largest subnet mask as the DIP will to forwared to that interface

For 2nd:

since number of 1's in subnet mask = 17 our subnet mask for this will be 255.255.128.0 

145.36.109.70
AND
255.255.128.0
=145.36.0.0
since it does not match with subnet address so it can't be the answer 

FOR 3rd

since number of 1's in subnet mask = 18 our subnet mask for this will be 255.255.196.0 

145.36.109.70
AND
255.255.196.0
=145.36.64.0
It matches with the subnet address so the answer CAN be E3 but we still need to check further!

 
For 4th: 

                  
                  
 

                  
since number of 1's in subnet mask = 24 our subnet mask for this will be 255.255.255.0


                  
145.36.109.70
AND
255.255.255.0
=145.36.109.0

                  

                   Since it doesn't match with the subnet address so it cannot be the answer.
                   

                   

                   Therefore finally I can conclude that Answer is E3 that is option A as it is the longest subnet mask.
                   

                   

                   Note that Default is used when DIP doesnot match with any subnet address.
