A subnetted Class $B$ network has the following broadcast address: $144.16.95.255$

Its subnet mask

  1. is necessarily $255.255.224.0$
  2. is necessarily $255.255.240.0$
  3. is necessarily $255.255.248.0$
  4. could be any one of $255.255.224.0$, $255.255.240.0$,$255.255.248.0$

Ans: Option D

Given, Class B network with DBA (HID Part has 1's) = 144.16.95.255

144.16.01011111.11111111

Generally, in Class B, 16bits are HID, but if the network is subnetted, then we borrow bits from the HID.

$\underline{\text{DBA's Possible}}$              $\underline{\text{Subnet Mask}}$ (put zero's in HID part)

144.16.$\underbrace{010}_\text{SID}$ $\underbrace{11111.11111111}_\text{HID}$ => 255.255.224.0

144.16.$\underbrace{0101}_\text{SID}$ $\underbrace{1111.11111111}_\text{HID}$ => 255.255.240.0

144.16.$\underbrace{01011}_\text{SID}$ $\underbrace{111.11111111}_\text{HID}$ => 255.255.248.0

144.16.$\underbrace{011011}_\text{SID}$ $\underbrace{11.11111111}_\text{HID}$ => 255.255.252.0
......

since they haven't given information about the number of subnets then any of the above are possible. Hence Option D

Option ($D$) is correct. In the broadcast address for a subnet, all the host bits are set to $1$. So as long as all the bits to the right are $1$, bits left to it can be taken as possible subnet.

Broadcast address for subnet is $.95.255$   $.0101 1111. 1111 1111$ (as in Class $B$, $16$ $bits$ each are used for network and host)

So, we can take minimum $3$ $bits$ (from left) as subnet and make rest as host bits(as they are $1$).

$.224.0$     $1110 0000. 0000 0000$   (leftmost $3$ $bits$ for subnet)

$.240.0$     $1111 0000. 0000 0000$  (leftmost $4$ $bits$ for subnet)

$.248.0$     $1111 1000. 0000 0000$  (...          $5$ $bits$ for subnet )

144.16.95.255 - Given The Address is in Class B hence first 16 bit or first 2 octets determine Network Id and remaining Host Bits.

Given It is a Broadcast Address and we know in Broadcast address all the Host Bits are 1.

.95.255 –  01011111 . 11111111

But in this all the 16 Host Bits are not 1 there are some 0's (Zeroes) as well it means some host bits has been used for Subnet Id. Now we need to identify which one are used for Subnet bits and which one are used for Host bits.

The One with 0 (1st and 3rd bit in 3rd octet from left) are definetly part of Subnet Id. But what about the bits which are 1 are they part of Subnet bits or Host bits so this we don't know.

So, Our goal is to get subnet mask

As we know to get Subnet Mask make all the Network bits as 1 and all the other bits as 0.

Hence,

255.255.1_1_ _ _ _ _ _ _ . _ _ _ _ _ _ _ _ _

So the empty bits can be either part of host bits or subnet bits.


.224.0 - 11100000 . 00000000

.240.0 - 11110000 . 00000000

.248.0 - 11111000 . 00000000

Many Other Possible Subnet Masks are also possible like :

.160.0 – 10100000.00000000

.252.0 – 11111100. 00000000 etc.

Hence all the Options are possible Subnets. Hence Option D) is the Correct Answer.

As it is a DBA, so there must be continuous 1s at the end.

So here are 13 continuous 1s.

And to get the first ID, i.e., NID, we need to replace all these 1s with 0s.

So NID for 144.16.95.255 = 144.16.01000000.0 = 144.16.64.0

So when we perform given IP Address AND Subnet mask in Options, we get 144.16.64.0 every time as NID.



\[
\begin{align*}
144.16.95.255 \text{ AND } 255.255.224.0 &= 144.16.64.0 \\
144.16.95.255 \text{ AND } 255.255.240.0 &= 144.16.64.0 \\
144.16.95.255 \text{ AND } 255.255.248.0 &= 144.16.64.0 \\
\end{align*}
\]
