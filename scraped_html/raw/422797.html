The baseline execution time of a program on a $2 \mathrm{GHz}$ single core machine is $100$ nanoseconds ( $n s)$. The code corresponding to $90 \%$ of the execution time can be fully parallelized. The overhead for using an additional core is $10 ~ns$ when running on a multicore system. Assume that all cores in the multicore system run their share of the parallelized code for an equal amount of time.

The number of cores that minimize the execution time of the program is __________.

The code that cannot be parallelized runs for 10ns.

For every additional core, additional 10ns overhead is added to total execution time, overhead for n cores is $(n-1)10$ns.

The code that can be parallelized runs for 90ns on one core, it runs for $\frac{90}{n}$ns on n cores.

Let t(n) be the execution time, it is given by -

$t(n) = 10 + (n-1)10 + \frac{90}{n} = 10n + \frac{90}{n}$

$t(1) = 100, t(2) = 65, t(3)  = 60, t(4) = 62.5, t(5) = 68, ...$

Answer - 3

Baseline execution time is 100 ns.

Program that can be parallelized is 90% i.e. 90 ns program.

Overhead cost of additional core is 10 ns.

 

Execution time (for n core) =    $\frac{Parallelized\,time}{n}$ + Fixed time + (overhead time $*$ (n - 1))

t(n) = $\frac{90}{n}$ + 10 + 10 $*$ (n-1)

t(n) = $\frac{90}{n}$ + 10n

 

To minimise t(n), derivative of t(n) = 0

\begin{align*}
&\Rightarrow -\frac{90}{n^2} + 10 = 0 \\
&\Rightarrow 10 = \frac{90}{n^2} \\
&\Rightarrow n^2 = 9 \\
&\Rightarrow n = \sqrt{9} \\
&\Rightarrow n = 3
\end{align*}

 

Hence, Answer = 3

Why Overhead is (N−1)⋅O(N - 1) \cdot O(N−1)⋅O:

  • NNN represents the total number of cores being used, not the number of additional cores.
  • The overhead is incurred only for additional cores beyond the first one because the first core is already part of the baseline execution and does not introduce any additional coordination overhead.
  • Thus, if NNN cores are used, there are N−1N - 1N−1 additional cores, each contributing overhead OOO.

Revised Explanation of Overhead

If NNN total cores are used:

  1. The first core has no overhead.
  2. Each of the remaining N−1N - 1N−1 cores introduces an overhead OOO.

The total overhead becomes:

Total Overhead=(N−1)⋅O\text{Total Overhead} = (N - 1) \cdot OTotal Overhead=(N−1)⋅O

If you were to instead phrase it as "using NNN additional cores", then the total number of cores would be N+1N + 1N+1, and overhead would be recalculated accordingly.

In this problem, the question explicitly assumes NNN total cores, so the overhead is correctly modeled as (N−1)⋅O(N - 1) \cdot O(N−1)⋅O.

2 GHz Machine implies Cycle-Time of 0.5 ns or you can say for every 1 ns, there are 2 cycles

Now, Given Exec Time of 100 ns implies 200 cycles out of which 90% i.e. 180 cycles can be parallelized

Task is to distribute 180 cycles equally in 'x' number of cores (each 2 GHz) such that exec time gets minimized

Now 

Take x=2, which means 20ns(Overhead) + 90 cycles(each core i.e. 180 cycles/2) i.e. 45ns totaling to 65ns

Take x=3 which means 30ns(Overhead) + 60 cycles(each core i.e. 180 cycles/3) i.e. 30ns totaling to 60ns

Take x=4 which means 40ns(Overhead) + 45 cycles(each core i.e. 180 cycles/4) i.e. 22.5 ns totaling to 62.5ns

Take x=5 which means 50 ns(Overhead) + 36 cycles(each core i.e. 180 cycles/5) i.e. 18 ns totaling to 65 ns

Clearly, minima can be observed at x=3 i.e. 60 ns

The baseline execution time of a program on a 2GHz single core machine is 100 nanoseconds.
" single core machine " This refers to it being a non pipelined system 
The cycle time can be calculated via ,  cycle time =  1/(clock rate )  which is 1/2ghz = 0.5 ns 
The program/code we are referring to gets fully executed in 100 ns 
So the  number of cycles involved in execution of program is  = total time /  1 cycle time 
which is  =  100 ns / 0.5 ns =  200 cycles
 

The code corresponding to 90% of the execution time can be fully parallelized. 
This statement here refers to the fact that the code getting executed in 90 % of the 100 ns time which is 90 ns ,  in 90ns the number of cycled executed would be = 90/ 0.5 = 180 cycles
so of the 200 cycles required for full execution of the program , 180 can be parallelized or  pipelined
 

Now, 
Heading into pipelining the 180 cycles
for that we need to know 3 things for the execution time 
1 what are the number of segments in the pipeline ( k) 
2 what is the cycle time of execution at a segment  ( Pipeline time )
3 Number of instructions ( n )
And thus with these we can write the execution time =  (k+n-1) tp 

In this case k and n both are unknown 
lets take an example of a pipelined instruction
 

cycle 1cycle 2cycle 3cycle 4cycle 5cycle 6
Instuction fetchinstruction decodeoperand fetchexecution Write back 
 Instuction fetchinstruction decodeoperand fetchexecution Write back

Assume all of them were getting executed in 1 cycle time each so we can observe in 2 instances of running them parallely we were able to run 10 cycles worth of instructions with pipelining in 6 cycle time only .
Here in our problem we are doing for 180 cycles
How are k and n can be related 

As we saw above in order to run 10 cycles on 5 segmented( k) pipeline number of instances ( n)  was number of cycles / number of segments in pipeline
 

So  n here = 180 / k
 

So let our pipeline be of k segments
Then ,  time of execution would be 
10 ns for each segment overhead = 10*k ns 
For 180 pipelined cycles , ( k + n -1 ) * tp = (k + 180 / k - 1) * 0.5 ns
For rest 20 non pipelined istruction time = 20* 0.5 ns 

Execution time of the program is thus =
10*k ns + (k + 180 / k - 1) * 0.5 ns + 20* 0.5 ns = E(k)
minimize E(k) by doing d(E(k))/d(k)  
d(E(k))/d(k)  = `0 = K^2 = 60/7
K = 2.92  == 3

Hence correct answer is 3.

\[
\textbf{Amdahl's Law:} \quad
\text{Speedup} = \frac{T_{\text{old}}}{T_{\text{new}}} = \frac{1}{(1 - F) + \frac{F}{N}}
\]

\[
\text{For multicore systems with overhead, the modified formula becomes:}
\]

\[
\text{Speedup} = \frac{1}{(1 - F) + \frac{F}{N} + (N - 1) \times \text{Overhead}}
\]

\[
T_{\text{new}} = T_{\text{old}} \times \left[ (1 - F) + \frac{F}{N} \right] + (N - 1) \times \text{Overhead}
\]

\[
\text{The } (N - 1) \text{ term appears because the first core does not add overhead — only the additional cores do.}
\]

\[
T_{\text{old}} = 100\,\text{ns}, \quad F = 0.9, \quad \text{Overhead} = 10\,\text{ns}
\]

\[
T_{\text{new}} = 100 \times \left[ (1 - 0.9) + \frac{0.9}{N} \right] + (N - 1) \times 10
\]

\[
\begin{aligned}
N = 1 &: \quad T_{\text{new}} = 100 \times (0.1 + 0.9) + 0 = 100\,\text{ns} \\
N = 2 &: \quad T_{\text{new}} = 100 \times (0.1 + 0.45) + 10 = 65\,\text{ns} \\
N = 3 &: \quad T_{\text{new}} = 100 \times (0.1 + 0.3) + 20 = 60\,\text{ns} \\
N = 4 &: \quad T_{\text{new}} = 100 \times (0.1 + 0.225) + 30 = 62.5\,\text{ns}
\end{aligned}
\]

\[
\boxed{
\begin{array}{c|c}
\text{Number of Cores (N)} & T_{\text{new}}\,(\text{ns}) \\
\hline
1 & 100 \\
2 & 65 \\
3 & 60 \\
4 & 62.5 \\
\end{array}
}
\]

\[
\text{Minimum Execution Time: } T_{\text{new}} = 60\,\text{ns at } N = 3
\]

The requirement is to minimize the time and for this minimized time we need to find the number of cores.
Note that the total time of execution in this new system varies as you vary the number of cores within the system.
This clearly indicates the approach to be used to solve this question. Its none other than the "maxima and minima" concept covered by sachin sir within the lectures.

let 't' be the total time of execution of program P in this system and let 'n' be the number of cores used.
$ \text{Total execution time} = \frac{90ns}{\text{number of cores}} + 10 * \text{additional cores} + \text{remaining time}
                                    = \frac{90}{n} + 10*(n-1) + 10 $

90% of the execution time i.e. 90ns can be parallelized and each core runs their share for same amount of time. Remaining time i.e. 10ns is not shared.

Just to understand the intuition lets take the example.
Suppose we had 10 such cores; so 90ns shared and eventually each core would have taken 9ns. Since these 10 cores run parallely, so after execution starts, the execution of 90% of the program happens in exactly 9ns (plus there are some overheads).

According to above equation,
$ t = \frac{90}{n} + 10*(n-1) + 10$
Since we need to minimize the execution time, we will differentiate this equation wrt n and then put dt/dn = 0 ....(refer sachin sir's calculus playlist for getting intuition on such type of questions)

Differentiating both sides wrt n, we get
$ \frac{dt}{dn} = \frac{-90}{n^2} + 10 $

Put dt/dn = 0
$ ∴0 = \frac{-90}{n^2} + 10 $

On solving we get n = 3
