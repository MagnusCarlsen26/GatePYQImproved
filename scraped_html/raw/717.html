Suppose the adjacency relation of vertices in a graph is represented in a table Adj $(X,Y).$ Which of the following queries cannot be expressed by a relational algebra expression of constant length?

  1. List all vertices adjacent to a given vertex
  2. List all vertices which have self loops
  3. List all vertices which belong to cycles of less than three vertices
  4. List all vertices reachable from a given vertex

The answer is D.

  1. This is a simple select query.
  2. This is the simple query we need to check $X=Y$ in the where clause.
  3. Cycle $< 3$ . Means cycles of lengths $1$ and $2$. The cycle of length $1$ is easy., the same as self-loop. The cycle of length $2$ is also not too hard to compute. Though it'll be a bit more complex, will need to do like $(X,Y)\; \&\; (Y, X )$ both present and $ X != Y$. We can do this with a constant length (not depending on the number of tuples) RA query.
  4. This is the hardest part. Here we need to find closure of vertices. This will need a kind of loop. If the graph is like a skewed tree, our query must loop for $O(N)$ times. We can't do this with a constant length query here.

Answer: D.

\[ \text{Adj} = \{ (A, B), (A, C), (C, D), (C, A), (D, D) \} \] 

below is the rln Adj corresponding to the above graph

\[ \begin{array}{c|c} \textbf{X} & \textbf{Y} \\ \hline A & B \\ A & C \\ C & D \\ C & A \\ D & D \\ \end{array} \]   

Each row says "x is adjacent to y." This is sometimes called an adjacency list representation in database design, and it’s used both for storing graphs and for hierarchical structures 

A. List all vertices adjacent to a given vertex \[ \pi_{Y} \bigl( \sigma_{\text{Adj.X} = \text{vertex}} (\text{Adj}) \bigr) \]

B. List all vertices which have self loops \[ \pi_{X} \bigl( \sigma_{\text{Adj.X} = \text{Adj.Y}} (\text{Adj}) \bigr) \]   Explanation:
Since self loops are edges from a vertex to itself, this query selects all rows where X = Y, then projects the vertex (X), correctly listing vertices with loops (like D in your example)

C. List all vertices which belong to cycles of less than three vertices \[ \pi_{X,Y} \Bigl(   \sigma_{(\text{Adj.X} = \text{Adj.Y}) \,\lor\,            ((\text{Adj.X} \neq \text{Adj.Y}) \land (\text{Adj.Y} = \text{Adj.X}))}   (\text{Adj}) \Bigr) \]   Explanation:                                                                                                                                cycles of vertices less than 3 is self loop and cycle b/w two vertices

D. List all vertices reachable from a given vertex here basically we gotta find adjacent of vertex recursively eg. A adj to C, C adj to D , A is also adj to b so, basically we are traversing either BFS or DFS would work, and in my knowledge there is no such relational algebra querry to list all reachable vertices from a vertex.

A. List all vertices adjacent to a given vertex (e.g., vertex a)

  • Expression:

  • πY(σX=′a′(Adj))

  • Explanation:
    This is a simple selection of all edges originating from a, followed by projection to get adjacent vertices. The expression length is constant (does not depend on graph size).

  • Example:
    For Adj = {(a,b), (a,c), (b,a)}, query for a returns {b, c}.

B. List all vertices with self-loops

  • Expression:
    πX(σX=Y(Adj))

  • Explanation:
    A self-loop occurs when a vertex has an edge to itself. The expression filters edges where X = Y and projects the vertex. The length is constant.

  • Example:
    For Adj = {(a,a), (b,c), (c,c)}, result is {a, c}.

C. List all vertices in cycles of length < 3

  • Expression:
    πX(σX=Y(Adj))∪πX(ρAdj1(X1,Y1)(Adj)⋈X1=Y2∧Y1=X2ρAdj2(X2,Y2)(Adj))

  • Explanation:

    • Part 1: Vertices in self-loops (cycle length = 1).

    • Part 2: Vertices in 2-cycles. The join finds pairs (u,v) and (v,u) (indicating a 2-cycle), and projects u and v.

    • The expression length is constant (fixed joins/projections).

  • Example:

    • Graph: a → b → a (2-cycle), c → c (self-loop).

    • Self-loops: {c}.

    • 2-cycles: Join finds (a,b) and (b,a) → projects {a, b}.

    • Result: {a, b, c}.

D. List all vertices reachable from a given vertex (e.g., vertex a)

  • Why it cannot be expressed with constant length:

    • Transitive closure requires recursively following paths of arbitrary length.

    • Relational algebra without recursion has fixed operators (e.g., joins, unions).

    • To compute reachability:

      • Start with vertices adjacent to a.

      • Then vertices adjacent to those, etc.

      • This requires repeated joins (number depends on graph diameter).

    • Example:

      • Path: a → b → c → d.

      • To find d is reachable, we need 3 joins:
        Adj⋈Adj⋈Adj(on Y = X).

      • For longer paths, more joins are needed.

    • Since the graph size is arbitrary, no fixed number of operations suffices for all cases.
