​​​​​Consider a network that uses Ethernet and $IPv4$. Assume that $IPv4$ headers do not use any options field. Each Ethernet frame can carry a maximum of $1500$ bytes in its data field. A UDP segment is transmitted. The payload (data) in the UDPO segment is $7488$ bytes.

Which ONE of the following choices has the CORRECT total number of fragments transmitted and the sixe of the last fragment including $IPv4$ header?
  1. $5$ fragments $1488$ bytes
  2. $6$ fragments $88$ bytes
  3. $6$ fragments $108$ bytes
  4. $6$ fragments $116$ bytes

MTU $= 1500$ B
It is given that IPv4 headers do not use any options field, that means IPv4 Header Size is $20$ B.
IP Header Size $= 20$ B

So, MTU without IP Header $= (1500 - 20)$ Bytes $= 1480 $ Bytes

Now, Payload for UDP Segment is $7488$ Bytes.
UDP Header is of $8$ Bytes.

So, 
Payload for IP datagram $=$ Payload for UDP segment $+$ UDP Header.
$= 7488 + 8 = 7496$

Now, fragmenting this Payload for IP Datagram using MTU without IP Header.

Total Fragments $= 6$
Last packet data without header $= 96$ Bytes
Last packet size with IP Header $= 96 + 20 = 116$ Bytes.

So, Total no of fragments are $\mathbf{6}$ and size of last packet including IP Header is $\mathbf{116}$ Bytes.

Answer: $\mathbf{D}$

MTU (Maximum Transmission Unit) refers to the maximum amount of payload data that the data link layer can carry in a single frame.

It does not include the overhead added by the data link layer itself (such as Ethernet headers and trailers).

For example, if the MTU of an Ethernet link is 1500 bytes, this means that up to 1500 bytes of Layer 3 (Network Layer) data.

The data link layer will then add its own headers and trailers on top of this payload, resulting in a total frame size that is larger than the MTU.

The Maximum Transmission Unit (MTU) of a link layer (e.g., Ethernet) is defined as the maximum size of the network-layer packet (e.g., IPv4 datagram) that can be carried in a single link-layer frame.

In other words:

  • MTU includes the entire IP datagram that is, IP header + IP payload.
  • It does not include the Ethernet header or trailer.
  • For standard Ethernet, MTU = 1500 bytes, meaning the IP datagram must be ≤ 1500 bytes.

This is critical: fragmentation occurs when the IP datagram exceeds the MTU of the outgoing link.

Given Data

UDP data size = 7488 bytes  
UDP header = 8 bytes  
IPv4 header = 20 bytes (no options)  
Link MTU = 1500 bytes

A common mistake is to assume only the “UDP data” is passed to IP. But the IP layer receives the entire UDP segment header plus data as its payload.

Thus, the IP payload = UDP header + UDP data =  
$$
8 + 7488 = 7496 \text{ bytes}
$$

The full IP datagram sent by the source is therefore:

$$
\text{IP Header (20 B)} + \text{IP Payload (7496 B)} = 7516 \text{ bytes}
$$

Since $7516 > 1500$, this datagram cannot be transmitted over Ethernet without fragmentation.

Complete Structure of the Original IP Datagram

The entire packet before fragmentation is structured as follows:

$$
\boxed{
\begin{array}{|c|c|c|}
\hline
\text{IPv4 Header} & \text{UDP Header} & \text{UDP Data} \\
\text{(20 bytes)} & \text{(8 bytes)} & \text{(7488 bytes)} \\
\hline
\end{array}
}
$$

This single 7516-byte IP datagram must be fragmented into smaller IP datagrams, each ≤ 1500 bytes in total size, to comply with the Ethernet MTU.

Fragmentation Details

Maximum IP datagram size per fragment = MTU = 1500 bytes  
IPv4 header = 20 bytes  
Maximum fragment payload = $1500 - 20 = 1480$ bytes  

Per IPv4 standards (RFC 791), the payload of every fragment except the last must be a multiple of 8 bytes (because the Fragment Offset field counts in 8-byte units).

Since $1480 \div 8 = 185$, 1480 is already a multiple of 8, so we use 1480-byte payloads for all non-final fragments.

Now fragment the 7496-byte IP payload:

Number of full 1480-byte fragments:  
$$
\left\lfloor \frac{7496}{1480} \right\rfloor = 5 \quad (\text{since } 5 \times 1480 = 7400)
$$

Remaining bytes:  
$$
7496 - 7400 = 96 \text{ bytes}
$$

Thus, total fragments = 6

  • First 5 fragments: 1480-byte payload → total size = $1480 + 20 = 1500$ bytes  
  • Last fragment: 96-byte payload → total size = $96 + 20 = \boxed{116}$ bytes

 

 

$$
\color{lime} \boxed{\text{D. 6 fragments, 116 bytes}}
$$
